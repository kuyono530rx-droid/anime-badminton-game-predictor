# Anime Character Badminton Tournament Predictor

This project simulates a badminton tournament between anime characters.

## Features

- Fetch character information from the Jikan (MyAnimeList) API
- Use an LLM to estimate badminton attributes (here is OpenAI API)
- Simulate rallies, games, matches, and tournaments
- Monte Carlo simulation to generate rankings
- Local caching to avoid repeated API calls

## How It Works

1. Character information is retrieved from the Jikan API.
2. An LLM estimates attributes such as:
   - Power
   - Agility
   - Stamina
   - Technique
   - Decision
   - Mental
   - Consistency
3. Matches are simulated using probability models.
4. Multiple tournaments are simulated to produce a ranking.

## Run

Set your OpenAI API key.

PowerShell:
```
$env:OPENAI_API_KEY="your_key_here"
```
macOS/Linux:
```
export OPENAI_API_KEY=your_key_here
```

## How the su=imulation works
run_ranking -> for each simulation -> simulate_tournament -> while bracket has more than 1 player -> for each match in this round -> simulate_match -> while no one has won 2 games -> simulate_game -> while game not finished -> simulate each rally

## Notes

- Results are based on simulated badminton attributes rather than canonical combat strength.
- This project is mostly generated by AI, specifically Codex 5.4.
- Example characters already have cached attributes in character_stats_cache.json.
- To test other characters, edit the character list in badminton.py.
